    6. Què és un callback? Posa un exemple.

    La funció callback és una funció que es passa a una altra de manera semblant a un argument. Després és invocada per una funció externa per completar una acció.

    Exemple

    
            // La funció fnNom és el callback, que conté el nom d'una persona a l'hora d'iniciar sessió.

            function fnNom(nom) {
                console.log("Hola, sóc "+nom);
            }

            // La funció demanaNom fa que la pàgina web pregunti el nom de l'usuari i el nom s'enregistrarà a fnNom.

            function demanaNom(callback) {
                let nom = prompt("Hola. Com et dius?");
                callback(nom);
            }
            
            // S'activa la funció demanaNom amb fnNom com el seu paràmetre. D'aquesta manera, demanaNom pot accedir al paràmetre 
            // de fnNom i mostrar-lo.

            demanaNom(fnNom);

    7. Què és una promesa en JavaScript (promise)? Posa un exemple. Explica també la diferència entre la programació síncrona i asíncrona. Posa un exemple.

    Una promesa és un objecte utilitzat per computacions asíncrones que representa un valor.

    Exemple


    
            let promesa = new Promise((resoldre,rebutjar) => {
                
                // S'utilitza el paràmetre resoldre en cas que la tasca es realitzi amb èxit, i rebutjar si falla.
                // El setTimeout s'encarrega de simular codi asíncrona.
                
                setTimeout(function() {
                    // Es mostra aquest missatge si la tasca es fa amb èxit.
                    resoldre("resolt");
                }, 250); 
            });

            // S'activa la variable promesa mostrant el missatge d'èxit si la seva tasca s'ha finalitzat correctament.

            promesa.then((missatge) => {
                console.log("El procés està "+missatge);
            });
    A la programació síncrona, cadascuna de les instruccions espera a l'anterior per executar-se, mentre que a la programació asíncrona no existeix cap sincronia entre les instruccions per poder executar-se.
